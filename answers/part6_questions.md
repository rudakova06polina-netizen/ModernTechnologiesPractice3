# Часть 6: Контрольные вопросы

## 1) В каком порядке выполняются конструкторы при создании объекта подкласса?
Ответ: Сначала выполняется конструктор родителя, потом конструктор подкласса.

Ошибки: если у родителя нет конструктора по умолчанию, нужно явно вызвать super()

Пример:
```java
class Animal {
Animal() { System.out.println("1. Родитель"); }
}
class Cat extends Animal {
Cat() { System.out.println("2. Подкласс"); }
}
new Cat();
```
## 2) Можно ли переопределить `static` метод? Что такое method hiding?
Ответ:Нельзя переопределить static методы. Если написать такой же static метод в подклассе,
это называется "сокрытие метода" (method hiding). При сокрытии выбор метода зависит от типа переменной, а не от объекта.

Ошибки: static методы НЕ работают полиморфно как обычные.

Пример:
```java
class Parent {
    static void test() { System.out.println("Parent"); }
}
class Child extends Parent {
    static void test() { System.out.println("Child"); }
}
Parent p = new Child();
p.test();
```
## 3) Что произойдет, если в `switch` нет `default` и ни один `case` не совпал?
Ответ: Ничего не произойдет. Программа просто пропустит весь switch и продолжит выполнение
дальше. Ошибки не будет.

Ошибка: это не прямо ошибка, но не обязательно писать default

## 4) Почему в `for-each` нельзя удалять элементы коллекции? Как правильно?
Ответ: Потому что for-each внутри использует итератор, и при удалении элемента напрямую итератор ломается - возникает ошибка

Ошибка: при удалении элемента из коллекции во время цикла for-each, итератор "теряется" и выбрасывает ошибку.
```java
for (String s : list) { list.remove(s); }
```

Пример (правильный):
```java
list.removeIf(s -> s.equals("удалить"));
```
## 5) Чем отличается `break` с меткой от обычного `break`?
Ответ: Обычный break выходит только из текущего (самого внутреннего) цикла.
break с меткой выходит из всех вложенных циклов до указанной метки.

Ошибки: забыть двоеточие после метки 

## 6) Что означает `@Override`?
Ответ: @Override - это аннотация (подсказка компилятору), которая означает
"этот метод должен переопределять метод родительского класса".
Компилятор проверяет, что такой метод действительно существует у родителя.
Если нет - выдает ошибку.

Ошибки: возможна  опечатка в имени метода

Пример:
```java
@Override
public void write() {
    System.out.println("пиши");
}
```
## 7) Какой модификатор доступа выбрать для видимости в подклассах?
Ответ: protected. Он дает доступ:
- внутри своего класса
- внутри подклассов (даже в других пакетах)
- внутри классов того же пакета

Ошибки: использовать private для полей, которые нужны в подклассе

## 8) Разница между `final` классом и `sealed` классом?
Ответ: final класс - от него нельзя наследоваться
sealed класс - от него можно наследоваться только указанным классам.

Ошибки: думать, что sealed и final - одно и то же.

## 9) Можно ли иметь несколько конструкторов?
Ответ: Да, это называется перегрузка конструкторов. Конструкторы отличаются:
- количеством параметров
- типами параметров
- порядком параметров

Ошибки: создавать два конструктора с одинаковыми параметрами.

Пример:
```java
String color;
Car() { this("белый"); }
Car(String color) { this.color = color; }
}
```
## 10) Что такое ковариантный тип возвращаемого значения?
Ответ: Это возможность переопределенного метода возвращать более конкретный тип, чем метод родительского класса.

Ошибки: думать, что тип возврата должен быть точно таким же

```java
class Animal {
Animal get() { return new Animal(); }
}
class Cat extends Animal {
@Override
Cat get() { return new Cat(); }  // возвращаем Cat, а не Animal
}
```
